# ADR-001: Arquitectura Hexagonal para aislar el dominio

## Estado

Aceptado — 2026-05-17

## Contexto

El sistema de pedidos debe soportar multiples tipos de procesamiento y notificaciones sin acoplar el dominio a Spring o JPA.

## Decision

Se adopta Arquitectura Hexagonal utilizando puertos e implementaciones en infraestructura y adaptadores.

## Consecuencias

### Positivas

- Dominio desacoplado.
- Facil mantenimiento.
- Mejor testabilidad.

### Negativas

- Mayor numero de clases.
- Curva de aprendizaje mas alta.
